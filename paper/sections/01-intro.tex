\section{Introduction}
\label{sec:intro}

The DHCD dataset~\cite{acharya2015dhcd} is the standard benchmark for
46-class handwritten Devanagari character recognition. Top-1 accuracies have
crowded into a narrow band above 99.7\,\%: MallaNet~\cite{malla2025} reaches
99.71\,\% with a \textbf{17.32\,M}-parameter convolutional architecture;
Mishra~\cite{mishra2021} independently reports 99.72\,\% with a similarly large
model. At this level successive gains are measured in \emph{hundredths of a
percent}. Models still carry parameter budgets far beyond what a compact
on-device recognizer can sustain.

Whether those gains are real is a question existing DHCD papers leave open.
Point accuracies dominate. McNemar testing, when it appears, is applied
selectively against weaker baselines without exact $p$-values. With
$n{=}13{,}800$ test samples near 99.7\,\%, raw-accuracy rankings alone carry
little inferential weight.

The decisive question is \emph{paired}: do two models fail on
\emph{different} examples? We answer it with exact McNemar tests for every
student--teacher pair and Wilson confidence intervals for each model's accuracy.
Applying this systematically changes what the leaderboard actually says.

A \textbf{1.11\,M}-parameter student reaches statistical parity with the
prior state-of-the-art~\cite{malla2025} (McNemar $p=0.345$)---and the
same parity holds \emph{even without knowledge distillation}.
Across distillation targets (TTA-softened vs.\ clean no-TTA, and a supervised
control) and teacher ensemble size (9 vs.\ 15 teachers), \emph{no lever}
produces a statistically significant improvement. Every configuration tested
converges to the same \textbf{11-error intrinsic floor}; meaningful progress
now requires a harder evaluation set or substantially different modelling
assumptions.
