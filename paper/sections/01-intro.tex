\section{Introduction}
\label{sec:intro}

The DHCD dataset ~\cite{acharya2015dhcd} is a standard benchmark for 46 classes of handwritten Devanagari character recognition. The top-1 accuracy is concentrated in a narrow band above 99.7\,\%. MallaNet~\cite{malla2025} reaches 99.71\,\% in the convolutional architecture with \textbf{17.32\,M} parameters. Mishra ~\cite{mishra2021} independently reported 99.72\,\% for a similarly large model. At this level, continuous gain is measured at \emph{hundredths of a
percent}. The model still has a parameter budget that far exceeds what a compact on-device recognizer can maintain.

Whether these benefits are genuine remains an open question in existing DHCD papers. Point accuracy dominates. When McNemar test appears, it is applied selectively against a weaker baseline without having an exact $p$ value. $n{=}13{,}800$ When test samples are close to 99.7\,\%, raw accuracy rankings alone have little inferential weight.

The crucial issue is \emph{paired}. Do the two models fail in the \emph{different} example? The exact McNemar test across all student-teacher pairs answers directly, with Wilson confidence interval bounding the accuracy of each model. Once this discipline is applied, the leaderboard will look different.

A \textbf{1.11\,M}-parameter student reaches statistical parity with the
prior state-of-the-art~\cite{malla2025} (McNemar $p=0.345$). That holds
\emph{even without knowledge distillation}. Neither the distillation target
(TTA-softened versus clean) nor the ensemble size (9 versus 15 teachers)
makes a statistically significant difference. Every configuration tested
converges to the same \textbf{11-error intrinsic floor}; progress now
requires a harder evaluation set or substantially different modelling
assumptions.
